3:T2e33,<p><a href="https://github.com/koocyton/open-rust-claw">基于 rust 的 telegrame LLM 一个消息意图分类器 </a></p>
<p><a href="https://github.com/koocyton/rust-openclaw/releases/download/latest/intel-mac-rust-bot">download rust-openclaw for intel mac </a></p>
<p><a href="https://github.com/koocyton/rust-openclaw/releases/download/latest/arm64-rust-bot">download rust-openclaw for arm64 linux </a></p>
<p>一个 Rust 编写的 Telegram Bot 常驻进程，监听 Telegram 频道/群组/私聊消息，通过 LLM（大语言模型）智能判断用户意图：对于提问直接回答，对于操作指令自动转化为 shell 命令并执行，支持返回图片。</p>
<h2>架构</h2>
<pre><code class="hljs language-scss">Telegram 频道/群组消息
       │
       ▼
  ┌──────────┐
  │ rust-bot │  (常驻进程，Long Polling)
  └──────────┘
       │
       ▼
  ┌──────────────────┐
  │ LLM API 意图分类  │  (OpenAI 兼容接口)
  └──────────────────┘
       │
       ├── 问题 → 直接回答，编辑消息返回
       │
       └── 操作 → 解析命令列表
                    │
                    ▼
              ┌──────────┐
              │ Shell 执行 │
              └──────────┘
                    │
                    ▼
              结果回传 Telegram（文本 + 图片）
</code></pre>
<h2>功能特性</h2>
<ul>
<li><strong>智能意图识别</strong> — LLM 自动判断消息是提问还是操作指令</li>
<li><strong>问答模式</strong> — 提问类消息直接由 LLM 回答</li>
<li><strong>命令执行</strong> — 操作类消息自动生成 shell 命令并执行</li>
<li><strong>图片支持</strong> — 命令输出中包含图片路径时自动发送图片（如截图）</li>
<li><strong>消息编辑</strong> — 「正在分析...」状态消息会被结果直接覆盖，不刷屏</li>
<li><strong>并发处理</strong> — 多条消息同时处理，不排队阻塞</li>
<li><strong>频道支持</strong> — 同时支持私聊、群组和频道消息</li>
<li><strong>详细日志</strong> — 每步操作带时间戳和耗时统计，方便排查问题</li>
</ul>
<h2>工作流程</h2>
<ol>
<li><strong>启动</strong> — 读取 <code>config.toml</code>，清理 Webhook，启动 Long Polling</li>
<li><strong>监听</strong> — 通过 Telegram Bot API 长轮询接收消息（支持频道/群组/私聊）</li>
<li><strong>分析</strong> — 发送「🔄 正在分析...」，调用 LLM 判断意图</li>
<li><strong>处理</strong>
<ul>
<li>提问 → LLM 直接回答，编辑覆盖状态消息</li>
<li>操作 → 解析命令列表 → 显示执行计划 → 逐条执行（失败时可自动修正重试）→ 回传结果</li>
</ul>
</li>
<li><strong>图片/视频</strong> — 如果命令输出中包含图片或视频文件路径，自动发送到频道</li>
</ol>
<h2>快速开始</h2>
<h3>前置条件</h3>
<ul>
<li>Rust 1.70+</li>
<li>一个 Telegram Bot Token（从 <a href="https://t.me/BotFather">@BotFather</a> 获取）</li>
<li>一个 OpenAI 兼容的 LLM API（如 <a href="https://openrouter.ai">OpenRouter</a>）</li>
</ul>
<h3>构建</h3>
<pre><code class="hljs language-bash">cargo build --release
</code></pre>
<h3>OpenWrt ARM64 交叉编译（MTK FILOGIC 820 / MT7981B 等）</h3>
<p>目标为 <strong>aarch64-unknown-linux-musl</strong>，与 OpenWrt 常见 musl 环境兼容。</p>
<p><strong>方式一：使用 <a href="https://github.com/cross-rs/cross">cross</a>（推荐，无需本机装 ARM 工具链）</strong></p>
<p>需先安装并启动 Docker。然后：</p>
<pre><code class="hljs language-bash">cargo install cross
rustup target add aarch64-unknown-linux-musl
./scripts/build-openwrt.sh
<span class="hljs-comment"># 或直接：</span>
cross build --target aarch64-unknown-linux-musl --release
</code></pre>
<p>产物在 <code>target/aarch64-unknown-linux-musl/release/rust-bot</code>，脚本会复制到 <code>./openwrt-out/rust-bot</code>。若不用 cross 而直接用 <code>cargo build --target ...</code>，需本机具备 aarch64-linux-musl 的 C 工具链（如 musl-cross 或 OpenWrt SDK），否则 <code>ring</code> 等 C 依赖会编译失败。</p>
<p><strong>方式二：使用 OpenWrt SDK 工具链</strong></p>
<p>在 <code>.cargo/config.toml</code> 中为 <code>[target.aarch64-unknown-linux-musl]</code> 设置 <code>linker</code> 为 SDK 中的 <code>aarch64-openwrt-linux-musl-gcc</code> 的路径，然后：</p>
<pre><code class="hljs language-bash">rustup target add aarch64-unknown-linux-musl
cargo build --target aarch64-unknown-linux-musl --release
</code></pre>
<p><strong>上传与运行</strong></p>
<p>将 <code>rust-bot</code> 和 <code>config.toml</code> 拷到设备（如 <code>/opt/rust-bot/</code>），在 OpenWrt 上需有 libc（musl 已静态链接进 musl 目标的可执行文件，一般无需额外库）。直接运行：</p>
<pre><code class="hljs language-bash">./rust-bot /path/to/config.toml
</code></pre>
<h3>配置</h3>
<pre><code class="hljs language-bash"><span class="hljs-built_in">cp</span> config.example.toml config.toml
<span class="hljs-comment"># 编辑 config.toml，填入你的 Bot Token 和 LLM API 配置</span>
</code></pre>
<h3>运行</h3>
<pre><code class="hljs language-bash"><span class="hljs-comment"># 使用默认配置文件 config.toml</span>
./target/release/rust-bot

<span class="hljs-comment"># 指定配置文件路径</span>
./target/release/rust-bot /path/to/config.toml

<span class="hljs-comment"># 开启 debug 日志</span>
RUST_LOG=debug ./target/release/rust-bot
</code></pre>
<h3>Telegram Bot 设置</h3>
<ol>
<li>在 <a href="https://t.me/BotFather">@BotFather</a> 创建 Bot，获取 Token</li>
<li>如果用于<strong>频道</strong>：将 Bot 添加为频道管理员</li>
<li>如果用于<strong>群组</strong>：将 Bot 加入群组</li>
<li>启动程序，发送一条消息，从日志中获取 <code>chat_id</code></li>
<li>将 <code>chat_id</code> 填入 <code>config.toml</code> 的 <code>allowed_chat_ids</code></li>
</ol>

<h2>Skills（扩展技能）</h2>
<p>通过 <strong>skills</strong> 可以扩展 bot 能处理的问题类型，而不改主程序代码。每个 skill 是一个子目录，内含 <code>skill.toml</code>，定义名称、描述、给 LLM 的提示片段和安装说明。</p>
<ul>
<li><strong>启用</strong>：在项目根下建 <code>skills</code> 目录，按 <code>skills/README.md</code> 添加 <code>skill.toml</code>，可选在 <code>config.toml</code> 中设置 <code>skills_dir = "skills"</code>。</li>
<li><strong>列出技能</strong>：在 Telegram 中发送「有哪些技能」。</li>
<li><strong>安装方式</strong>：发送「怎么安装 截图」等，可查看对应技能的安装说明。</li>
</ul>
<p>详见 <a href="skills/README.md">skills/README.md</a>。</p>
<h2>错误修复与自动重试</h2>
<p>当某条命令执行失败时，程序会向 LLM 询问修正方式，并可从回复中解析出建议的命令自动重试：</p>
<ul>
<li><strong>配置项</strong> <code>executor.max_fix_retries</code>：单条命令失败后，最多请求 LLM 修正并重试的次数，默认 <strong>10</strong>。设为 <strong>0</strong> 时不做自动重试，仅把「解决建议」附在报告里给用户参考。</li>
<li><strong>流程</strong>：命令失败 → 将命令、退出码、stderr 发给 LLM（并注入相关 skill 上下文）→ 从回复中解析出修正命令（如代码块或反引号内的内容）→ 执行该命令；若仍失败则重复，直到成功或达到 <code>max_fix_retries</code>。全部重试后仍失败时，会在报告末尾再附一次「解决建议」。</li>
<li>这样在环境或参数可被 LLM 修正的情况下，有机会自动执行成功，而无需用户手动改命令再发一次。</li>
</ul>
<h2>LLM 接口</h2>
<p>本程序通过 OpenAI 兼容的 Chat Completions API（<code>/v1/chat/completions</code>）与 LLM 交互，不使用 MCP 协议。</p>
<p>支持的 LLM 服务：</p>
<ul>
<li><a href="https://openrouter.ai">OpenRouter</a> — 聚合多家模型</li>
<li><a href="https://platform.openai.com">OpenAI</a></li>
<li>任何兼容 OpenAI API 格式的服务（如 Ollama、vLLM、LocalAI 等）</li>
</ul>
<p>LLM 负责将用户消息分类为两种意图：</p>
<ul>
<li><strong>问题</strong> — 返回回答内容</li>
<li><strong>命令</strong> — 返回要执行的 shell 命令列表</li>
</ul>
