<h2>简短说明（最多 80 个字符）</h2>
<p>将短信与未接来电转发到 API 或邮箱，支持过滤、日志与后台常驻。</p>
<h2>详细说明</h2>
<p>短信转发可将您 Android 设备上的每一条短信和未接来电转发到您自己的 API（Webhook）或邮箱，便于自动化、留痕或在其他设备或服务器上使用短信。</p>
<p><strong>主要功能</strong></p>
<p>• <strong>双通道转发</strong> — 可转发到 HTTP/HTTPS 接口（支持 GET/POST、自定义请求头与请求体），也可转发到任意 SMTP 邮箱。</p>
<p>• <strong>发送方与内容过滤</strong> — 仅转发指定号码或包含关键词的短信；留空则转发全部。</p>
<p>• <strong>未接来电转发</strong> — 可选择将未接来电一并转发（主叫号码为发送方，内容为「未接来电」）到同一 API 或邮箱。</p>
<p>• <strong>后台常驻</strong> — 以前台服务与常驻通知运行，在授予权限后即使关闭应用也会持续监听并转发。</p>
<p>• <strong>转发日志</strong> — 查看每次转发的成功/失败、支持重试失败项或清空记录。</p>
<p>• <strong>多配置</strong> — 可添加多个 API 与邮箱目标，随时启用或关闭任一配置。</p>
<p><strong>使用步骤</strong></p>
<ol>
<li>授予短信（及按需授予通话）权限。</li>
<li>添加一个或多个 API 或邮箱配置，如需未接来电通知可开启「转发未接来电」。</li>
<li>开启监听，应用将按您的规则转发符合条件的短信与未接来电。</li>
</ol>
<p><strong>使用要求</strong></p>
<p>• 具备短信功能的 Android 设备<br>
• 使用 API 时：您的接口需支持 HTTP/HTTPS<br>
• 使用邮箱时：SMTP 服务器信息（如 Gmail 需使用应用专用密码）</p>
