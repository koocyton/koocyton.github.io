<p>A replica of the classic <strong>SoundStream</strong> screensaver for modern macOS, built with Objective-C and OpenGL.</p>
<h2><a href="https://github.com/koocyton/soundstream_mac">https://github.com/koocyton/soundstream_mac</a></h2>
<h2>About / 关于</h2>
<p><strong>English:</strong></p>
<p>SoundStream was a beloved screensaver that visualized audio input as flowing particle streams. This project recreates that visual effect as a native macOS <code>.saver</code> bundle using OpenGL particle rendering.</p>
<p>However, due to macOS sandboxing restrictions on screen savers, <strong>the new macOS screensaver cannot access the microphone</strong>. Apple's ScreenSaver framework runs <code>.saver</code> bundles in a sandboxed process (<code>legacyScreenSaver</code>) that does not grant microphone permissions — and there is no way to request them. As a result, this version uses autonomous particle animation instead of reacting to audio input.</p>
<p><strong>中文：</strong></p>
<p>SoundStream 是一款经典的屏保程序，它能将音频输入可视化为流动的粒子流。本项目使用 Objective-C 和 OpenGL 粒子渲染，将这一视觉效果复刻为原生 macOS <code>.saver</code> 屏保包。</p>
<p>但由于 macOS 对屏保的沙盒限制，<strong>新版 macOS 屏保无法访问麦克风</strong>。Apple 的 ScreenSaver 框架在沙盒进程（<code>legacyScreenSaver</code>）中运行 <code>.saver</code> 包，不会授予麦克风权限，也无法主动申请。因此，本版本使用自主粒子动画，而非响应音频输入。</p>
<hr>
<h2>Features / 功能特性</h2>
<ul>
<li>Smooth 60fps particle stream animation / 流畅的 60fps 粒子流动画</li>
<li>HSL color cycling with glow effects / HSL 色彩循环与辉光效果</li>
<li>OpenGL hardware-accelerated rendering with multisampling / OpenGL 硬件加速渲染，支持多重采样</li>
<li>Automatic Retina display support / 自动支持 Retina 显示屏</li>
<li>Particle trail with fade-in/fade-out lifecycle / 粒子拖尾，带有淡入/淡出生命周期</li>
<li>Stream head with edge-aware steering and braking / 流头具有边缘感知转向与制动</li>
</ul>
<hr>
<h2>Why No Microphone? / 为什么无法使用麦克风？</h2>
<p><strong>English:</strong></p>
<p>Starting from macOS Catalina (10.15), screen savers run inside a heavily sandboxed host process. The sandbox profile does not include microphone (<code>com.apple.security.device.audio-input</code>) entitlements. Even if you add <code>NSMicrophoneUsageDescription</code> to <code>Info.plist</code>, the system will never prompt the user for permission — the audio session simply fails silently.</p>
<p>This is a known limitation that affects all third-party <code>.saver</code> bundles. The original SoundStream used Quartz Composer (<code>.qtz</code>), which had broader system access in older macOS versions.</p>
<p><strong>中文：</strong></p>
<p>从 macOS Catalina (10.15) 开始，屏保在严格沙盒化的宿主进程中运行。沙盒配置不包含麦克风（<code>com.apple.security.device.audio-input</code>）权限。即使在 <code>Info.plist</code> 中添加了 <code>NSMicrophoneUsageDescription</code>，系统也不会弹出权限请求对话框——音频会话会静默失败。</p>
<p>这是影响所有第三方 <code>.saver</code> 包的已知限制。原版 SoundStream 使用 Quartz Composer（<code>.qtz</code>），在旧版 macOS 中拥有更广泛的系统访问权限。</p>

<h2>Installation / 安装</h2>
<p><strong>English:</strong></p>
<ol>
<li>Build the project in Xcode (or use the pre-built <code>soundstream.saver</code>)</li>
<li>Double-click <code>soundstream.saver</code> to install</li>
<li>Open <strong>System Settings → Screen Saver</strong> and select <strong>soundstream</strong></li>
</ol>
<p><strong>中文：</strong></p>
<ol>
<li>在 Xcode 中构建项目（或使用预编译的 <code>soundstream.saver</code>）</li>
<li>双击 <code>soundstream.saver</code> 进行安装</li>
<li>打开 <strong>系统设置 → 屏幕保护程序</strong>，选择 <strong>soundstream</strong></li>
</ol>
